XShell 是一款功能强大的安全终端模拟软件，它可以在 Windows 界面下访问远端不同系统下的服务器，实现远程控制的目的。无论是运维工程师、开发人员还是学生，都经常需要用它来连接 Linux 服务器。

虽然 XShell 是商业软件，但它提供了“家庭/学校免费版”，功能完整，足以满足个人学习和非商业用途。本教程将手把手教你完成 XShell 8 的下载、安装、激活、配置以及连接服务器，全程“保姆级”指导，让你轻松上手。

#### 一、XShell 是什么？
XShell 是一个终端模拟器，支持 SSH1、SSH2、SFTP、TELNET、RLOGIN 等多种协议。通过它，你可以在 Windows 电脑上方便地连接和管理 Linux/Unix 服务器，执行命令、传输文件。

**核心特点**：

*   **安全性高**：支持 SSH 协议，保障数据传输安全。
*   **功能强大**：支持标签页、会话管理、脚本自动化、文件传输（配合 Xftp）等。
*   **界面友好**：提供丰富的配色方案和样式选择，可自定义程度高。
*   **免费版够用**：家庭/学校免费版功能完整，仅限制同时打开的会话数量（最多4个）。

#### 三、安装 XShell 8
1.  双击运行下载好的 `.exe` 安装文件。

2.  在安装向导中，点击“下一步”。

3.  勾选“我接受许可证协议中的条款”，然后点击“下一步”。

4.  选择安装路径。默认是 C 盘，为了系统整洁，建议点击“浏览”更改到其他盘（如 D 盘），然后点击“下一步”。

5.  选择程序文件夹，可以直接使用默认的“Xshell 8”，点击“下一步”。

6.  点击“安装”开始安装。

7.  等待安装进度条完成，然后点击“完成”。

#### 四、激活免费版许可
1.  首次启动 XShell 8，会弹出一个“免费许可”窗口。

2.  输入你的姓名和邮箱地址（与下载时填写的邮箱一致），然后点击“提交”。

#### 五、创建会话并连接服务器
1.  启动 XShell，会自动弹出“会话”窗口。如果没有弹出，可以点击左上角的“新建”按钮。

2.  在“新建会话属性”窗口中，填写以下信息：

    *   **名称**：填写一个便于你识别的名称，例如“我的阿里云服务器”。
    *   **协议**：选择 `SSH`。
    *   **主机**：填写服务器的公网 IP 地址。
    *   **端口号**：默认是 `22`。

3.  点击左侧的“用户身份验证”。

    *   **方法**：选择 `Password`。
    *   **用户名**：填写服务器的用户名，通常是 `root`。
    *   **密码**：填写服务器的登录密码。

4.  点击“确定”保存会话。

5.  在“会话”列表中，选中你刚刚创建的会话，然后点击“连接”。

6.  如果是第一次连接，可能会弹出一个“SSH 安全警告”，询问是否接受主机密钥，点击“接受并保存”即可。

7.  稍等片刻，如果一切配置正确，你就会看到服务器的命令提示符，表示连接成功！

#### 六、常用配置与技巧
**解决中文乱码**
如果连接后中文显示为方框或乱码：

1.  在会话列表中，右键点击你的会话，选择“属性”。

2.  在左侧选择“终端”。

3.  在右侧的“编码”下拉菜单中，选择 `UTF-8`。

4.  点击“确定”保存。

**复制与粘贴**
*   **复制**：直接用鼠标左键在终端窗口中划选文本，就会自动复制到剪贴板。
*   **粘贴**：在终端窗口中点击鼠标右键，即可粘贴剪贴板中的内容。

**注意**：不要使用 `Ctrl+C` 和 `Ctrl+V`，因为在 Linux 终端中，`Ctrl+C` 通常是中断命令。

**会话管理**
你可以创建多个会话，方便管理不同的服务器。在“会话管理器”中，可以新建文件夹，将会话分组，例如分为“生产环境”和“测试环境”。

#### 七、常见问题解答
**Q1：连接时提示“Connection refused”怎么办**？
*   **检查服务器 IP 和端口**：确认填写的 IP 地址和端口号（默认22）是否正确。
*   **检查服务器 SSH 服务**：确认服务器的 SSH 服务是否正在运行。
*   **检查防火墙**：确认服务器的防火墙是否放行了 22 端口。

**Q2：免费版和商业版有什么区别**？
*   **功能**：免费版功能完整。
*   **限制**：免费版最多只能同时打开 4 个会话标签页。
*   **用途**：免费版仅限个人非商业用途。

**Q3：提示“The remote SSH server rejected X11 forwarding request.”警告**？
这个警告不影响正常使用，可以忽略。如果想消除它，可以在会话属性的“SSH”->“隧道”中，取消勾选“X11 转发”。
